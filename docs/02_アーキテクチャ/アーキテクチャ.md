# アーキテクチャ

本アプリケーションは、バックエンドサーバーを必要とせず、Webブラウザのみで動作する **クライアントサイドWebアプリケーション** として構築します。

## システム構成

- **フロントエンド**
  - ユーザーインターフェース（UI）の描画およびデータのライフサイクル管理のすべてを担当します。
  - **React** と **Vite** (必要に応じてTypeScript) を用いて、ブラウザ上で動作するSPA（Single Page Application）として構築します。
  - 画面遷移（新規作成、編集、一覧表示）やユーザー操作の受け付けを行います。
- **データストレージ**
  - バックエンドサーバーの代わりに、ブラウザの組み込みストレージ（`localStorage` または `IndexedDB`）を使用してデータを永続化します。
  - これにより、サーバーレスでオフライン環境でも動作可能な設計とします。

## 主要な依存関係 (Dependencies)

開発および実行に必要な主なライブラリ・ツールは以下の通りです。

### アプリケーション依存
- **`ajv`** (必須): JSON Schema (`todo.schema.json`) に基づくデータの厳密なバリデーションを行うために利用します。データの保存前にクライアント側で検証します。

### 開発時依存
- **UIフレームワーク / ビルドツール**: **React** と **Vite** を採用し、SPAの開発効率とユーザー体験を向上させます。
- **スタイリング**: **Tailwind CSS** を採用し、ユーティリティファーストなアプローチで効率的かつ一貫性のあるデザインを構築します。
