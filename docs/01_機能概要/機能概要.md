# 機能概要

本アプリケーションは、`todo.schema.json` に記載されたデータ形式を用いてToDoを管理するアプリです。

## UI レイアウト

- 画面左側に固定の**サイドバー**を配置し、各ビュー（リスト・カンバンボード・ガントチャート）を切り替えます。
- すべてのビューはブラウザの画面幅全体（`w-full`）を使用して表示されます。

## 提供する主な画面

以下の画面を提供します。

- **ToDoの新規作成画面**
  - タイトル（必須）、詳細な説明、ステータス、優先度、開始日時、終了日時、タグの入力が可能です。
- **ToDoの編集画面**
  - 既存のToDoの各項目（タイトル、説明、ステータス、優先度、開始日時、終了日時、タグ）の更新が可能です。
- **ToDoのリスト表示画面（List View）**
  - 登録されたタスクを優先度（高・中・低）や終了日時順にソートして表示します。
  - タグをクリックすることで、同じタグを持つタスクのみにフィルタリングできます。
  - タスク一覧の最下部の「Add New Task」ボタンから新規作成画面に遷移できます。
- **カンバンボード画面（Board View）**
  - タスクをステータス（To Do / In Progress / Done）ごとにカラムに分けて表示します。
  - タグをクリックすることで、同じタグを持つタスクのみにフィルタリングできます。
  - 全てのカラムの最下部に「Add Task」ボタンを設置しています。
  - 詳細については [カンバンボード機能.md](./カンバンボード機能.md) を参照してください。
- **ガントチャート画面（Gantt View）** ※開発予定
  - タスクの `start_date` と `end_date` をもとに、タスクの期間をタイムライン上のバーで可視化します。
  - 詳細については [ガントチャート機能.md](./ガントチャート機能.md) を参照してください。

## 主な機能仕様

- **タスクのステータス管理:** `todo`（未着手）, `in_progress`（進行中）, `done`（完了）の3段階で管理します。
- **優先度設定:** `low`（低）, `medium`（中）, `high`（高）の3段階で設定可能です。
- **日付管理:** 作成日時、更新日時、開始日時（`start_date`）、終了日時（`end_date`）をISO 8601形式で記録・管理します。
- **タグ付け:** 任意のタグを複数付与し、用途に応じた分類が可能です。タグクリックによるフィルタリングに対応しています。
