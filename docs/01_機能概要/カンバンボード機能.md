# カンバンボード機能の追加

## 概要

既存のリスト表示（`ListPage`）に加え、タスクのステータス（`todo`, `in_progress`, `done`）を視覚的に管理できる「カンバンボード（Board）」ビューを追加します。これにより、各タスクの進行状況を直感的に把握できるようになります。

## 追加する画面・コンポーネント

### 1. Boardビュー (`src/pages/Board.tsx`)

カンバンボードのメイン画面となるコンポーネントです。

- **構成**:
  - `To do`, `In progress`, `Done` の3つのカラム（列）を横並びに配置します。
  - 各カラムには該当するステータスのタスクカードを縦にリスト表示します。
  - カラムヘッダーにはステータス名と、そのステータスに属するタスクの件数を表示します。
- **カードの仕様**:
  - `ListPage` と同様の情報（タイトル、優先度、期日など）を表示しますが、カラム幅に合わせたコンパクトなデザイン（縦積みが基本）とします。
  - カードをクリックすることで、既存の「編集画面（`EditPage`）」へ遷移します。
- **レスポンシブ対応**:
  - 画面幅が広い（PC等）場合は3カラムを横並びにします。
  - 画面幅が狭い（スマートフォン等）場合はカラムを縦にスタック（1列表示）させるか、横スクロールで対応し、視認性を確保します。

### 2. メインルーティングの更新 (`src/App.tsx`)

ユーザーが「リストビュー」と「ボードビュー」を任意に切り替えられるようにします。

- **状態管理の拡張**:
  - `PageState` 型に、ボードビューを示す新しい状態 `| { type: 'board' }` を追加します。
- **ナビゲーションUI**:
  - アプリケーションのヘッダー部分に、ビューを切り替えるためのトグルボタン（またはタブ）を追加します。
  - 「List」と「Board」のアイコン（例: `List` アイコンと `Kanban` または `LayoutGrid` アイコン）を用いて、直感的に操作できるようにします。

## データ・状態管理への影響

- **ストレージ・スキーマ**:
  - `todo.schema.json` や `localStorage` に保存するデータ構造自体への変更は**不要**です。既存の `status` フィールドの値（`todo`, `in_progress`, `done`）をそのままカラムの振り分け条件として利用します。
- **ステータス更新**:
  - 初期の実装（フェーズ1）では、ドラッグ＆ドロップによるステータス変更は行わず、ボード上のカードをクリックして「編集画面（`EditPage`）」に遷移し、そこでステータスを変更する仕様とします。これにより、複雑な状態管理や外部ライブラリの導入を避け、安全かつ迅速に機能を提供します。

## 開発ステップ（フェーズ）

1. **ルーティング・ヘッダーUIの拡張**: `App.tsx` を改修し、空の `BoardPage` コンポーネントへの切り替え機能を作成。
2. **カラムレイアウトの実装**: `BoardPage` に3つのカラムを定義し、レスポンシブなCSS設定（Flexbox/Grid）を適用。
3. **データ流し込みとカードUI**: `localStorage` から取得したToDoデータをステータスごとにフィルタリングし、各カラムにタスクカードとして描画。カードクリック時の編集画面遷移を実装。
