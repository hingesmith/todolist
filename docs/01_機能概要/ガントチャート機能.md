# ガントチャート機能の追加

## 概要

タスクの `start_date`（開始日時）と `end_date`（終了日時）をもとに、タスクの期間をタイムライン上のバーとして可視化する「ガントチャート（Gantt）」ビューを追加します。これにより、複数タスクの期間・進行状況を一覧で直感的に把握できるようになります。

## 追加する画面・コンポーネント

### 1. Ganttビュー (`src/pages/Gantt.tsx`)

ガントチャートのメイン画面となるコンポーネントです。

- **タイムライン軸 (X 軸)**:
  - 表示期間の開始・終了を、登録されている全タスクの `start_date` / `end_date` から自動算出します。
  - ヘッダー行にはスケール（日・週・月）に応じた日付ラベルを表示します。
  - 今日の日付には**赤い縦ライン**を表示し、現在位置を一目で把握できるようにします。
  - タイムスケールを「Day / Week / Month」で切り替えるトグルボタンを設けます。

- **タスクバー (Y 軸)**:
  - `start_date` と `end_date` の両方が設定されているタスクのみをバーとして描画します。
  - バーの色はステータスで色分けします（To Do: 青、In Progress: 赤、Done: 緑）。
  - バーにホバーするとタスク名・開始日・終了日のツールチップを表示します。
  - バーをクリックすると既存の「編集画面（`EditPage`）」へ遷移します。

- **日付未設定タスクの表示**:
  - `start_date` または `end_date` が欠けているタスクは、ガントバーとして描画できないため、チャート本体の**下部に一覧**として表示します。
  - 各リストアイテムをクリックすることで編集画面に遷移し、日付を設定できます。

### 2. メインルーティングの更新 (`src/App.tsx`)

- `PageState` 型に `| { type: 'gantt' }` を追加します。
- 左サイドバーに「Gantt View」のナビゲーションボタンを追加します（`GanttChartSquare` アイコンを使用）。

## データ・状態管理への影響

- **ストレージ・スキーマ**: `todo.schema.json` への変更は**不要**です。既存の `start_date` / `end_date` フィールドをそのまま利用します。
- **新規フィールドの追加**: `start_date` および `end_date` は既に追加済みのため、対応不要です。

## 開発ステップ（フェーズ）

1. **ルーティング・サイドバーの拡張**: `App.tsx` を改修し、空の `GanttPage` コンポーネントへの切り替え機能を作成。
2. **タイムライン計算ロジックの実装**: タスクの日付範囲から表示期間・各バーの幅と位置（%）を計算するロジックを実装。
3. **チャートUIの実装**: ヘッダー（日付軸）・タスクバー・Today ライン・日付未設定タスク一覧を描画。
4. **タイムスケール切替の実装**: Day / Week / Month ビューの切り替えとそれに応じたヘッダー・グリッドの変更。
