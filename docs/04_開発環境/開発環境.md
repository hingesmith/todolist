# 開発環境

本アプリケーションは、**React** と **Vite** を用いたフロントエンドSPAとして構築されています。バックエンドサーバーは不要で、ブラウザ内で完結する設計となっています。

## 前提条件 (Prerequisites)

開発を始める前に、ご使用のOSに以下のソフトウェアがインストールされていることを確認してください。

- **Node.js** (必須: v24.14.0 LTS)
  - ダウンロード・インストール方法: [Node.js公式サイト](https://nodejs.org/) からインストーラーを取得するか、各種パッケージマネージャー（`nvm`, `fnm`, `volta` やOSの標準パッケージマネージャー）をご利用ください。
- **npm** (Node.js に同梱されています) または **yarn**, **pnpm** 等のパッケージマネージャー

---

## 開発環境のセットアップ (Setup)

1. **プロジェクトディレクトリに移動します。**

   このプロジェクトのルートディレクトリ(`todolist/`)をカレントディレクトリとします。

   ```bash
   cd path/to/todolist
   ```

2. **依存ライブラリのインストール**

   以下のコマンドを実行し、必要なパッケージ（React, Vite, ajv, Tailwind CSS など）をインストールします。

   ```bash
   npm install
   ```

---

## 開発用サーバーの起動 (Running the Dev Server)

開発中は、コードの変更を即座に画面に反映させるため（HMR: Hot Module Replacement）、専用の開発サーバーを立ち上げます。

```bash
npm run dev
```

起動後、コンソールに表示されるローカルURL（例: `http://localhost:5173/`）にブラウザでアクセスすると、アプリケーションの画面を確認できます。
※サーバーを停止するには、ターミナルで `Ctrl + C` を入力します。

---

## プロジェクト構造の概要

開発時に頻繁に触れるディレクトリの役割です（[アーキテクチャ.md](../02_アーキテクチャ/アーキテクチャ.md) も併せて参照してください）。

- `/src/pages` : 新規作成・編集・一覧などの各画面のメインビュー。
- `/src/components` : 再利用可能なUIパーツ（ボタン、フォーム部品など）。
- `/src/validation` : `todo.schema.json` と `ajv` を連携させたバリデーションロジック。
- `/src/storage` : （※今回はブラウザのデフォルト機能やメモリを利用するため、永続化に関するラップ処理などを置きます）

---

## プロダクションビルド (Build for Production)

本番環境向けにファイルを最適化（圧縮、バンドル化）する場合は以下のコマンドを実行します。

```bash
npm run build
```

実行後、`/dist` ディレクトリ内に最適化された静的ファイル群（HTML, CSS, JS）が出力されます。このディレクトリの中身を任意のWebサーバー（Apache, Nginx, Vercel, Netlify など）に配置することで、本番として公開可能です。
