# データモデル

本アプリケーションで扱うToDoデータは、プロジェクト直下にある `todo.schema.json` の内容に従って管理されます。

## データ仕様 (todo.schema.json)

`todo.schema.json` はToDoアイテムの構造を定義するJSON Schemaです。アプリケーション内のデータ登録、編集、および保存時には、このスキーマによるバリデーションを行い、データの整合性を担保します。

主な制約とデータ型は以下の通りです：

- **`id`** (必須): UUID形式の文字列。タスクの一意な識別子。
- **`title`** (必須): 1文字以上の文字列。タスク名。
- **`description`**: 文字列。詳細な説明（任意）。
- **`status`** (必須): `"todo"`, `"in_progress"`, `"done"` のいずれか。デフォルトは `"todo"`。
- **`priority`**: `"low"`, `"medium"`, `"high"` のいずれか。デフォルトは `"medium"`。
- **`created_at`** (必須): ISO 8601形式の日時文字列。
- **`updated_at`**: ISO 8601形式の日時文字列。
- **`due_date`**: ISO 8601形式の日時文字列（期限）。
- **`tags`**: 文字列の配列。要素は重複不可 (`uniqueItems: true`)。

## ライフサイクルとバリデーション

- **新規作成:** ユーザーが入力したデータに対し、フロントエンド側でJSON Schemaに基づく検証（例：`ajv`を利用）を実施し、必須項目(`id`, `title`, `status`, `created_at`)や型定義を満たしているか確認します。
- **編集:** 既存データの更新時にも、同様にスキーマを満たすことを保証します。
- **保存:** ブラウザのローカルストレージ（`localStorage` または `IndexedDB`）へ永続化するデータは、常にこのスキーマに完全に準拠した配列（`type: "array"`）形式として保存（および読み込み）されます。
